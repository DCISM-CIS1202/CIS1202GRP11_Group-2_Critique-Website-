    A university’s website serves as a crucial hub for students, faculty, and staff. Recognizing the importance of accessibility and inclusivity, the University of San Carlos has a responsibility to ensure that its website (ISMIS) is accessible to all users, including those with disabilities. This section aims to evaluate and analyze the website’s accessibility and its specific areas of concern.



  • ALT ATTRIBUTES

    • When a screen reader comes across null alt text, it will completely skip over the image, without ' announcing its presence. If no alt attribute is present, the screen reader will read the file name for the image instead, which can be a major distraction to those using screen-reading technology (SiteImprove, 2024). A specific number of images on ISMIS lack meaningful and appropriate alt text. Many use generic placeholders like “img-circle” or are entirely devoid of alt attributes. This is a critical issue, especially with regards to accessibility, since the alt attribute or alt text is essential for screen reader users to understand the purpose and content of the images.


    Figure 14-15: Home Page: Image has no alt attribute and inappropriate alt attribute
    Click

    Without these descriptive alt text, screen reader users will be unable to comprehend the information conveyed by the images, hindering their ability to navigate and understand the website. Moreover, images often serve as visual cues for navigation. Without alternative text, screen reader users cannot determine the destination of the link, making navigation difficult and confusing. This can lead to a significant loss of information, a frustrating user experience, and could create a significant barrier for students with visual impairments who rely on screen readers.

  • NAVIGATION USING KEYBOARD

    • Although the website can be navigated with a keyboard, it is very limited to how much the user can navigate and interact with the website. Using the different arrow keys such as the Up arrow and the Down arrow can make the website go up or down, respectively. Pressing Shift+Tab can select the different buttons in the website, and pressing Enter can activate the selected buttons or elements.

    However, it does not allow the user to select any of the dropdown menus so the interactive content becomes very limited to just the different links or buttons to the other announcement pages. So if the user wants to select “Student Services” which is a dropdown menu, they can not do so with a keyboard alone.


  • WCAG COMPLIANCE

    • Web Content Accessibility Guidelines (WCAG) 2.1 defines how to make Web content more accessible to people with disabilities. Accessibility involves a wide range of disabilities, including visual, auditory, physical, speech, cognitive, language, learning, and neurological disabilities (w3.org, 2024).

    Several elements exhibit insufficient color contrast between text and background, making them difficult to read for users with visual impairments. This can cause eye strain and make it challenging to distinguish between text and background elements. Refer to Figure 8 and 10 for reference.

ISMIS as a website is already pretty solid for its purpose and use. Its use and purpose is just to simply show students ways on how to avail for documents, get books, find information, and notify students of any important announcements.

However, there are still some features that need improvement and these are the following:

  1. Privacy Policy Dismissal – Allow students to opt out of seeing the privacy policy pop-up again after accepting it, even after logging out, to reduce repetitive interruptions.

  2. Navigation Bar Organization – Improve the UI structure by decluttering the navigation bar. The dropdown menus are currently disorganized, with some options appearing redundantly across multiple menus. Streamlining these sections will enhance usability.

  3. Interactive Enhancements – To make the interface more dynamic, consider adding subtle animations when users hover over dropdown menus. This would improve the visual appeal and user engagement.

  4. Mobile Responsiveness – Improve the website’s scaling on mobile devices. Some information is not fully displayed, affecting readability and navigation. Adjusting the layout for better responsiveness will create a smoother mobile experience.
